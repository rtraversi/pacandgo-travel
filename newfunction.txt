Cruise Deals Page + Admin Panel — Build Summary
                                                                                                                                         What was built

  Two interconnected systems for a travel advisor website:
  1. A client-facing deals page (deals.html) that displays cruise deals as clickable tiles with a modal detail view
  2. An admin panel (admin.html) with two management tools — one for cruise deals, one for the advisor's personal sailing schedule

  ---
  Tech stack

  - Pure static HTML/CSS/JS — no frameworks
  - Data stored in two JSON files fetched at runtime
  - Netlify Functions (Node.js) handle the admin backend
  - Claude API (claude-opus-4-7) rewrites raw cruise line emails into client-friendly deal cards
  - Saves are written back to GitHub via the GitHub Contents API, which triggers an auto-deploy on Netlify

  ---
  Files created

  Flat JSON array of deal objects. Each deal has:

    "cruiseLine": "Royal Caribbean",
    "ship": "Symphony of the Seas",
    "nights": 7,
    "departDate": "Aug 9, 2026",
    "homeport": "Galveston, TX",
    "destination": "Western Caribbean",
    "region": "caribbean",
    "ports": ["Cozumel", "Costa Maya", "Roatan"],
    "priceLabel": "Balcony from",
    "priceAmount": "$1,800",
    "pricePer": "/ 2 people",
    "priceNote": "Interior from $1,200",
    "teaser": "One-sentence hook shown on the tile",
    "description": "2-3 sentences shown in the modal",
    "inclusions": ["All meals", "Entertainment", "Pools & fitness"],

  Regions: caribbean, europe, mediterranean, alaska, other — used for filter chips.

  sailings-data.json

  The advisor's personal upcoming cruise schedule:

  deals.html

  Standalone page. Fetches deals-data.json via fetch() on load. Renders:
  - Filter chips by region at the top
  - Tile grid of deal cards (ship name, badge, teaser, price, nights, homeport)
  - Click any tile → modal with full description, ports, inclusions, cabin pricing, and a "Contact [Advisor Name]" CTA button
  - Design: ocean theme (navy #023e8a, coral #ff6b6b, sand/white), Playfair Display + DM Sans fonts

  admin.html

  Password-protected admin panel (login screen → admin screen). Two sections:

  Deals Manager:
  - Textarea to paste raw cruise line email text
  - "Extract Deals" button → calls Netlify function → Claude rewrites into deal objects → shows preview cards
  - Each extracted deal can be individually removed before adding
  - "Add Selected to Live Deals" merges into current deals list
  - "Save & Deploy" button → POSTs to Netlify function → commits deals-data.json to GitHub → Netlify auto-deploys
  - Current live deals listed with Remove buttons

  Our Cruises Manager (personal sailings):
  - Form to add a sailing: cruise line dropdown, ship name, departure date, nights, homeport, itinerary name, ports, badge checkboxes
  (New Ship, Icon-class)
  - Sailings table showing all entries sorted by date, past sailings dimmed
  - Remove button per row
  - "Save & Deploy" button → commits sailings-data.json to GitHub

  netlify/functions/admin-api.js

  Handles three actions (all POST, all password-protected):
  - extract-deals — sends email text to Claude with a system prompt that strips agent-only pricing language, converts to
  client-friendly copy, returns JSON array of up to 3 deals
  - save-deals — receives full deals array, commits deals-data.json to GitHub
  - save-sailings — receives full sailings array, commits sailings-data.json to GitHub

  Password check: compares password field in request body against ADMIN_PASSWORD env var, returns 401 if wrong.

  GitHub commit flow: GET the file to retrieve its SHA, then PUT with base64-encoded content + SHA.

  netlify.toml

  [build]
    publish = "."
    functions = "netlify/functions"

  [functions]
    node_bundler = "esbuild"

  [[redirects]]
    from = "/admin"
    to = "/admin.html"
    status = 200

  ---
  Claude system prompt (in the Netlify function)

  The prompt instructs Claude to:
  - Strip all travel-agent jargon (net rates, commissions, GTY, overrides)
  - Convert prices to per-couple estimates
  - Skip past sailings
  - Return up to 3 deals as a raw JSON array (no markdown wrapper)
  - Use friendly, honest language for leisure travelers

  ---
  Key design decisions

  - Sailings the advisor is personally going on should not appear on the client deals page — they're managed separately in
  sailings-data.json and shown as "Sail With Me" / "Our Cruises" on the main site
  - The admin login uses session storage so you stay logged in across page refreshes but not across browser sessions
  - The GitHub commit approach (instead of a database) keeps the site 100% static with zero hosting cost beyond Netlify's free tier
